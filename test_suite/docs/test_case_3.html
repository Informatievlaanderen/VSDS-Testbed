[TC3] Member discovery

The purpose of this test case is to ensure that a node from which all members of a collection can be discovered, can be found through a triple stating ex:C1 tree:view ex:N1 with ex:C1 being a tree:Collection and ex:N1 being a tree:Node.

This test case foresees the following steps:

Step Description
1 Data provider posts KBO dataset to the LDES Server (SUT) using the endpoint for ingestion (if not skipped).
2 The ITB fetches the processed data from the LDES server's access endpoint.
3 The ITB validates via SHACL shapes the returned data to ensure that all member nodes are discoverable.

 

Test data: The KBO BEL 20 dataset will be posted to your ingestion endpoint as part of the first step. If you don't want to do this per-test session you can do the setup separately before executing tests. In this case ensure that the relevant system configuration property is appropriately set to skip the automatic setup step.
