[TC1] Member discovery

The purpose of this test case is to ensure that a node from which all members of a collection can be discovered, can be found through a triple stating ex:C1 tree:view ex:N1 with ex:C1 being a tree:Collection and ex:N1 being a tree:Node.

This test case foresees the following steps:

Step Description
1 Data provider posts KBO dataset to the LDES Server (SUT) using the endpoint for ingestion (if not skipped).
2 The Testbed fetches the processed data from the LDES server's access endpoint.
3 The Testbed validates via SHACL shapes the returned data to ensure that all member nodes are discoverable. By making sure the current stream is a ldes:EventStream, also a tree:view existed in the ldes:EventStream.

 

Test dataset: The KBO BEL 18 dataset will be posted to your ingestion endpoint as part of the first step. If you don't want to do this per-test session you can do the setup separately before executing tests. In this case ensure that the relevant system configuration property is appropriately set to skip the automatic setup step.
