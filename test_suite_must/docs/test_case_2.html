[TC2] TreeNode to TreeCollection

The purpose of this test case is to ensure when the current page is a tree:Node, there MUST be a property linking the current page URL to the URI of the tree:Collection.

This test case foresees the following steps:

Step Description
1 Data provider posts KBO dataset to the LDES Server (SUT) using the endpoint for ingestion (if not skipped).
2 The Testbed crawls the whole graph of the input data from the LDES server's access endpoint. This step re-iterates for all configured fragmentation options: possibly Geospatial fragmentation, Time-literals fragmentation, and Strings fragmentation.
3 The Testbed validates the crawled data via SHACL shapes to ensure that when the current page is a tree:Node, there MUST be a property linking the current page URL to the URI of the tree:Collection.(this step re-iterates for all configured fragmentation options: possibly Geospatial fragmentation, Time-literals fragmentation, and Strings fragmentation

 

Test data: The KBO BEL 18 dataset will be posted to your ingestion endpoint as part of the first step. If you don't want to do this per-test session you can do the setup separately before executing tests. In this case ensure that the relevant system configuration property is appropriately set to skip the automatic setup step.
