[TC 6] No tree:path define fallback.

The purpose of this test case is to ensure that When no tree:path is defined, the tree:value MUST be compared to all members’ triples that can be compared to the tree:value as defined by the type of the relation (or when no members or collection are defined, on every triple in the page). When due to rdfs:range incompatibility, the object cannot be compared, the object will not be considered for comparison.

This test case foresees the following steps:

Step Description
1 Data provider posts KBO dataset to the LDES Server (SUT) using the endpoint for ingestion (if not skipped).
2 The ITB fetches the processed data from the endpoint of the LDES Server (by location).
3 The ITB crawls the fetched data and validates. The validation checks that when no tree:path is defined, the tree:value MUST be compared to all members’ triple.

 

Test data: The KBO BEL 20 dataset will be posted to your ingestion endpoint as part of the first step. If you don't want to do this per-test session you can do the setup separately before executing tests. In this case ensure that the relevant system configuration property is appropriately set to skip the automatic setup step.
