[TC 7] String fragmentation MUST be compared according to case-sensitive unicode ordering.

The purpose of this test case is to ensure that when the comparator relations such as tree:GreaterThanRelation is used in the string comparision, the strings MUST then be compared according to case-sensitive unicode ordering.

This test case foresees the following steps:

Step Description
1 Data provider posts KBO dataset to the LDES Server (SUT) using the endpoint for ingestion (if not skipped).
2 The Testbed crawls the whole graph of the input data from the LDES server's access endpoint of Strings fragmentation.
3 The Testbed validates the crawled data as one graph. The validation checks that when the comparator relations such as tree:GreaterThanRelation is used in the string comparision, the strings MUST then be compared according to case-sensitive unicode ordering.

 

Test data: The KBO BEL 18 dataset will be posted to your ingestion endpoint as part of the first step. If you don't want to do this per-test session you can do the setup separately before executing tests. In this case ensure that the relevant system configuration property is appropriately set to skip the automatic setup step.
