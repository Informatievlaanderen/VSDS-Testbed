[TC3] tree:Relation’s tree:value

The tree:Relation’s tree:value SHOULD be set.

This test case foresees the following steps:

Step Description
1 Data provider posts KBO dataset to the LDES Server (SUT) using the endpoint for ingestion (if not skipped).
2 The Testbed fetches the processed data from the LDES server's access endpoint.
3 The Testbed crawls the fetched data and validates via SHACL shapes the returned data to ensure that all tree:Relations have a tree:value.

 

Test data: The KBO BEL 18 dataset will be posted to your ingestion endpoint as part of the first step. If you don't want to do this per-test session you can do the setup separately before executing tests. In this case ensure that the relevant system configuration property is appropriately set to skip the automatic setup step.
